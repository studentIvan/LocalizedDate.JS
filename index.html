LocalizedDate.JS

Simple client script helper for localize and convert server datetime for user timezone

Download this project as a .zip file Download this project as a tar.gz file

LocalizedDate.js features

  • Perfect time localization, userlike, e.g. "tomorrow at 5:43"
  • International localization - russian and english locales support
  • Autodetect user locale and select best locale for him (e.g. ua -> ru)
  • Automatical converting datetime from server timezone to user timezone
  • Compatible with MySQL Datetime format

Constructor

Constructor create any LocalizedDate instance and give it to variable.

var ldt = new LocalizedDate(TIMEZONE, [LOCALE]);
  • TIMEZONE - String. Server timezone offset in hours (e.g. 4 for Europe/Moscow).
  • LOCALE - String. User locale as 2 letters (ru, en). Will be detect automatically if missed.

Method "localize"

This method localize standart datetime strings by LocalizedDate object locale and server timezone offset.

ldt.localize(DATETIME, [USER_CURRENT_DATETIME]);
  • DATETIME - Simple datetime string (Y-m-d H:i:s or other readable format).
  • USER_CURRENT_DATETIME - String. Using for debug only. Miss this parameter.

Simple code example

var ldt = new LocalizedDate(4); // timezone Europe/Moscow, locale autodetect
console.log(ldt.localize('2012-04-23 00:40:29'));
